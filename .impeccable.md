## Design Context

### Users
Trois publics principaux servis par la même plateforme :
1. **Apprenants** (cohortes, sprints, workshops) — sessions longues dans les pages `/learning/…` et `/[module]`. Lecture de contenus Notion transformés en UI produit.
2. **Prospects marketing** — landing/hubs publics (`/`, `/hubs`, `/blog`). Première impression, doivent sentir "produit fini, pas site Notion".
3. **Membres gated** — accès privé via cookie `gate_key`, navigation profonde dans les hubs.

Usage mixte desktop + mobile. Sessions de lecture longues (15-45 min) pour les apprenants ; scan rapide pour les prospects. Site en français principalement.

### Brand Personality
**Premium. Éditorial. Signature jaune.**

Le projet vise la sensation "produit fini, pas site Notion" (cf. `docs/Design System Transition Plan (premium Front).md`). Cohérent avec la suite d'outils Impulsion (OS_Impulsion partage l'accent jaune amber `#ffb300`).

Tonalité : magazine/journal professionnel, pas trop corporate, pas gadget. Rythme éditorial, typographie marquée, composition variée. L'info prime, mais la présentation est léchée.

### Aesthetic Direction
- **Thème** : light par défaut (lecture diurne de contenu long), dark supporté pour certaines sections marketing immersives.
- **Signature couleur** : yellow/amber `#ffb300` préservé. Tinter les neutrals vers cette hue (chroma 0.005-0.010) pour cohésion subconsciente avec les outils Impulsion. Accent rare (règle des 10%).
- **Typographie** : **pairing distinctif obligatoire** (display + body). L'Outfit actuel est dans la reflex-font-list /impeccable ; à remplacer par un choix plus spécifique. Tabular-nums sur métriques et dates.
- **Motion** : framer-motion déjà en place, à calibrer — mouvement fonctionnel (entrées, transitions d'état), pas décoratif. Respect strict de `prefers-reduced-motion`.
- **Spacing** : échelle 4pt sémantique (déjà partiellement en place dans tokens.css). Container queries sur widgets Notion pour qu'ils s'adaptent au contexte.
- **Rendu Notion** : c'est la feature centrale. Chaque type de bloc doit avoir un rendu intentionnel (callout, toggle, quote, code, bookmark, image, table, columns). Pas de rendu "défaut Notion".

### Design Principles
1. **Notion devient UI** — le pipeline Notion→bundle→render doit produire du contenu qui a l'air conçu, pas copié-collé.
2. **Une seule voix visuelle** — marketing, hubs et learning partagent les tokens. Pas de palette marketing distincte de la palette learning.
3. **Rythme éditorial** — alternance dense/aéré, section bands, respiration. La monotonie visuelle tue la lecture longue.
4. **Hiérarchie par typographie** — le poids/la taille/le contraste portent la structure, pas les bordures colorées.
5. **Performance first** — Speed Insights en place, animations `transform`/`opacity` only, images via Cloudinary avec dimensions calibrées.

### Anti-patterns spécifiquement bannis (à migrer)
- ❌ `border-left: 4px` coloré sur callouts — tell AI absolue, présent dans 9 variantes de `callout[data-callout-tone="*"]` → à remplacer par typographie + icône + tint de fond subtil.
- ❌ Multiples gradients linéaires/radiaux "premium" qui se sentent génériques (info-card variants AI/exercise/story/warning, btn-hero.btn-primary, section-band, divider-pill radial) → rationaliser à 1 gradient signature maximum.
- ❌ Glassmorphism décoratif : `backdrop-filter` sur pills, flip-card-surface, widget-surface → réserver au command palette/overlay seulement.
- ❌ Reflex fonts : **Outfit** est dans la banlist /impeccable → remplacer.
- ❌ Lift effects (`translateY(-2px)` on hover) sur callout/bookmark/info-card/pill/btn — trop d'endroits, ça amoindrit l'impact de chaque lift. À réserver aux cartes cliquables "principales".
- ❌ Shadows décalées (`0 18px 80px rgba(...)`) — calibrer une échelle 3 niveaux.
- ❌ Double gradient text + glow box-shadow — chaque CTA premium combine plusieurs effets, effet "gaming" à retirer.
